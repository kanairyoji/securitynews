カテゴリ: カテゴリ1「地政学リスク・国家サイバーオペレーション」
対象日: 2026年06月13日

総評

2026年6月13日、中国系脅威アクター「Velvet Ant」による10年間にわたる重要インフラへの潜伏侵害「Operation Highland」が詳細報告された。認証スタックそのものを乗っ取る高度な手法を用い、孤立ネットワーク（エアギャップ環境）への侵入を実現した点は、国家レベルの持続的作戦能力を改めて示すものだ。またCrowdStrikeの2026年技術脅威ランドスケープ報告では、国家支援型侵入の58%以上が中国系アクターによるものと記録されており、テクノロジーセクターへの集中的な標的化が継続していることが確認された。AI輸出規制をめぐる米政府とAnthropicの対立も国家サイバー戦略と交差する形で進展した。

1. Velvet Ant「Operation Highland」— 10年間の認証乗っ取りスパイ作戦

出典: BleepingComputer（2026-06-13）

  • 攻撃者: 中国系サイバースパイ Velvet Ant（「Sygnia」が分析）
  • 侵入期間: 2016年〜2026年（約10年間）
  • 標的: 大規模組織のエアギャップ重要インフラネットワーク
  • 主要TTP:
  • インターネット接続サーバーを経由してSOCKS5プロキシを構築
  • NginxとFastCGI（fcgiwrap）を改ざんし、HTTP POSTリクエスト経由でエアギャップ環境へのSSH実行パスを確立
  • Linux PAMモジュールをバックドア版に置換（9種類の亜種を特定）→ ハードコードパスワードで全セッション認証を制御
  • OpenSSHコンポーネント（ssh/sshd/scp）を改ざん版に置換 → 全ログイン・コマンド実行を記録
  • パスワード変更後も持続性維持、従来の封じ込め対策を無効化
  • 除去の困難さ: 多数の重要コンポーネントが改ざんされていたため、単純な削除ではシステム全体のロックアウトリスクがあった。Sygniaはテスト環境での検証とロールバック手順を整備してから本番クリーンアップを実施
  • 推奨対策: PAM・OpenSSH・Windows LSASSをEDR・FIM・MFAで継続監視。オフライン回復計画の整備

2. CrowdStrike 2026年技術脅威ランドスケープ報告 — 中国の野望が攻撃を加速

出典: CrowdStrike Blog（2026-06-14取得）

  • 国家支援型侵入の58%以上が中国系アクターに帰属（MURKY PANDA、MUSTANG PANDA、OVERCAST PANDA、SUNRISE PANDA、WARP PANDA など）
  • テクノロジーセクターへの集中的標的化：AI・知的財産・東南アジアのメールインフラ
  • MURKY PANDA：米国中心に340以上の組織にパスワードスプレー攻撃
  • 北朝鮮 FAMOUS CHOLLIMA：IT偽装就労でテックセクターへの侵入（全国家支援型作戦の47%）
  • STARDUST CHOLLIMA（北朝鮮）：週1億ダウンロードの Axios npm パッケージを侵害（サプライチェーン汚染）
  • eCrime勢力のデータリークサイト掲載：テクノロジー企業572社（他セクターを大幅超過）

国家サイバー戦略・法制度

カテゴリ: カテゴリ2「国家サイバー戦略・法制度」
対象日: 2026年06月13日
生成日時: 2026-06-14

総評

2026年6月13日の最大の出来事は、米商務省がAI企業Anthropicに対してFable 5とMythos 5モデルへの外国人アクセスを即時停止するよう命じた件だ。これはAI輸出規制という形での国家サイバー戦略の新局面を示している。モデルのジェイルブレイクを理由とした商用AIモデルの規制停止という前例のない手法であり、業界全体への波及が不可避だ。Anthropic側はこの措置の技術的根拠が不十分と反論しており、政府とAI企業の法的対立が深まっている。

1. 米政府、Anthropic Fable 5・Mythos 5 の外国人アクセスを輸出規制で停止命令

出典: BleepingComputer、The Hacker News、CyberScoop、SecurityWeek（2026-06-13）

経緯

  • 2026年6月12日 17:21 ET: 商務長官 Howard Lutnick 署名の書簡が Anthropic CEO Dario Amodei 宛に届く
  • 内容：Fable 5 および Mythos 5 への外国人アクセスを安全保障上の理由により「即時停止」
  • 対象：米国内外を問わずすべての外国国籍者（Anthropic 社員を含む）
  • Anthropic は外国籍者をリアルタイムで識別できないため、全ユーザー向けに両モデルをシャットダウン

背景・経緯

  • Fable 5は6月9日にリリースされ、6月22日まで Pro/Max/Enterprise 会員に無償提供中だった
  • Mythos 5は「Project Glasswing」経由で選別されたサイバーセキュリティ企業・政府機関（NSA含む）に提供
  • Fable 5のジェイルブレイク（コードベースを読ませて脆弱性を特定させる手法）が政府に報告されたとされる
  • Amazonの Andy Jassy CEO が政府に懸念を伝達したと複数メディアが報道
  • Trump AI 補佐官 David Sacks は「信頼できるパートナーがジェイルブレイクを提供し、Amodei氏は修正か停止を拒否した」と主張

Anthropic の反論

  • 「政府から受け取った証拠は口頭のみ。該当ジェイルブレイクは限定的・非普遍的で、GPT-5.5等の他モデルでも同様の能力が発揮できる」
  • 「完全なジェイルブレイク耐性はどのモデルプロバイダーにも不可能」
  • 「この基準を業界全体に適用すれば、すべてのフロンティアモデルの新規展開が実質的に停止する」
  • 「政府は危険なAI展開を停止する権限を持つべきだが、それは透明・公正・技術的根拠に基づくプロセスであるべき。今回の措置はその原則に沿っていない」

出典: BleepingComputer（2026-06-13）

封じ込めを困難にした要因

  • PAM（Linux Pluggable Authentication Modules）が改ざん版に置換 → 削除すると正規ユーザーもロックアウト
  • OpenSSH (ssh/sshd/scp) が改ざん版 → 即時削除でリモート管理が失効
  • 攻撃者はパスワード変更後も持続性を維持 → 通常の「パスワードリセット」手順が無効

Sygnia の除去アプローチ

  1. テストラボの構築: バイナリ置換プロセスを本番環境適用前に検証
  2. ホストプロファイリング: 各ホストの改ざん状況と依存関係を把握
  3. ロールバック手順の準備: 置換失敗時の回復方法を確立
  4. 段階的置換: 正規バイナリへの安全な置換を段階的に実施

再発防止策

  • PAM・OpenSSH・Windows LSASS を重要セキュリティ資産として扱いEDR・FIM・MFA・特権アクセス管理を適用
  • イミュータブルスナップショット付きバックアップの定期取得
  • バックアップとリカバリースクリプトは本番外の検証済みOS環境でテスト

1. なぜAIプロジェクトは停滞するのか — CIOの対応戦略

出典: CrowdStrike Blog（2026-06-14取得）

停滞の構造的原因

  • ビジネス部門がAIツールを提案 → 調達へ → セキュリティが懸念提起 → 法務・コンプライアンスが躊躇 → プロジェクト停止
  • 従来のセキュリティ・コンプライアンスレビューは決定論的システム（確定的入力・予測可能な出力）向けに設計されており、確率的・適応的・不透明なAIシステムには対応できていない

AIプロジェクトにおける主要懸念

  1. 外部・内部ホストAIモデルとの対話時の機密データ保護

1. Velvet Ant「Operation Highland」— 10年間の重要インフラスパイ活動

出典: BleepingComputer（2026-06-13）

活動概要

  • 帰属: 中国系サイバースパイ Velvet Ant（過去のF5 BIG-IP侵害・Cisco NX-OS ゼロデイ悪用でも確認）
  • 活動期間: 2016年〜2026年（約10年）
  • 標的: 大規模組織の重要インフラ部門（エアギャップネットワーク）
  • 目的: 長期的な諜報収集（intelligence collection）

諜報活動の特徴

  • 認証システムそのものを乗っ取ることで、すべての管理者ログイン・コマンド実行が監視可能な状態を達成
  • 「Velvet Ant was in a position to monitor every login, every command executed across compromised hosts」（Sygnia）
  • パスワード変更後も持続性を維持 → 従来の「侵害疑い→パスワードリセット」対応が機能しない
  • 9種類のPAM改ざんモジュールを別々のビルド環境でコンパイル → 高リソース・高組織性の国家アクター

過去の活動との連続性

  • 2024年：F5 BIG-IP デバイスを標的に3年間未検出のキャンペーン（Sygnia報告）
  • 2024年：Cisco NX-OS ゼロデイ悪用（Nexusスイッチ）
  • 2026年：上記と同一アクターが10年侵害の全容を報告

対諜報（Counter-Espionage）の観点

  • 事後的発見の困難さ: PAM・SSH改ざんは通常の監視では検出困難
  • 検出の鍵: ファイル整合性監視（FIM）によるシステムバイナリの継続的ハッシュ検証
  • 特権アクセス経路の分離: PAM・SSHへのアクセスそのものを制限されたジャンプホストに限定する設計

優先対応マトリクス

優先度 対象 理由
