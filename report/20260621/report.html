対象日: 2026年06月21日

総評

本日は国家背景の脅威アクターに関する年次レポート系の総括記事が目立った。CrowdStrike の 2026 年版テクノロジー脅威レポートは、国家支援型侵入の 58% 超を中国系アクターが占め、AI 技術・知財・サプライチェーンを狙う長期的なインテリジェンス収集が常態化していると指摘する。IT-ISAC／Food and Ag-ISAC の 2025 年脅威レポートでは Lazarus Group が両セクターで首位、ロシア系が脅威全体の約半数を占め、中国系は通信・クラウド・研究網への「事前展開（pre-positioning）」へ戦術をシフトしている。LOTL（環境寄生）手法はほぼ全アクターで採用が確認され、検知回避と長期潜伏が主流化している。

1. CrowdStrike 2026 テクノロジー脅威レポート — 中国の野心が攻撃を駆動

出典: CrowdStrike Blog（2026-06-21 取得）
- 2025/4〜2026/3 の観測で、国家支援型標的侵入の 58% 超が中国系アクターに帰属。MURKY PANDA・MUSTANG PANDA・OVERCAST PANDA・SUNRISE PANDA・WARP PANDA がテック業界を最重点で標的化。
- 動機は技術開発・知財・CCP のインテリジェンス収集目標との合致。中国の戦略的至上命題である「技術的自給自足」を背景に、AI 能力が高価値標的と評価。下流の顧客環境へのアクセス＝サプライチェーン侵害も狙う。
- SUNRISE PANDA は東・東南アジアのメール基盤（政府通信アクセスにつながり得る）、MURKY PANDA は 340 超の主に米組織へパスワードスプレー、WARP PANDA は北米テック企業で脆弱性悪用と持続的アクセスを継続。
- DPRK 系（FAMOUS／LABYRINTH／STARDUST CHOLLIMA）もテック業界を標的化。FAMOUS CHOLLIMA は対テック業界の国家支援型ハンズオン操作の 47% を占め、IT 労働者偽装による資金獲得を継続。

2. 2025 サイバー脅威ランドスケープ（IT／食料・農業セクター）

出典: Cyber Defense Magazine（2026-06-21）
- PASS（Predictive Adversary Scoring System）でアクターをスコア化。IT で 77、食料・農業で 72 のアクティブアクターを識別。
- Lazarus Group が両セクター首位（IT 89.0／食料農業 84.0）。IT は Sandworm（84.0）等の地政学的破壊型、食料農業は Qilin・Akira 等のランサム型と Dark Engine（76.0）等のハクティビストが台頭。
- 出自別ではロシア系が IT 脅威の 48.4%・食料農業の 59.3% を占め、中国系（IT 29%／食料農業 25.4%）は即時窃取より「事前展開」へ移行。Iran・North Korea は少数だが高能力。
- LOTL は両セクターで 100% のアクターが利用、96% 超がマルウェア改変で署名ベース検知を回避。

3. 中国系 FishMonger、SprySOCKS を Windows へ移植

出典: Security Affairs マルウェアニュースレター Round 102（2026-06-21）
- 中国系 FishMonger の兵器庫が更新され、SprySOCKS の Windows 版が登場。カーネルレベルのステルスと UEFI ブートキットの兆候が報告される。
- 同ニュースレターでは DPRK の APT37 が NarwhalRAT を MS テーマのフィッシングと dead-drop C2 で展開していることも取り上げられた。

4. 中国系アクターが医療研究ネットワークに 2 年間潜伏

出典: Security Affairs newsletter Round 582（2026-06-21）
- 公的・民間の医療コミュニティが、AI・サイバー・医療・国防研究を狙う China-Nexus アクターの標的に。研究網に約 2 年間潜伏していたと報じられた。国家による先端研究の長期窃取という典型的構図。

1. 大統領令 14409 後の AI セキュリティ — 次の一手

出典: CrowdStrike Blog（2026-06-21 取得）
- 2026/6/2 署名の EO 14409 は「セキュリティは AI の便益享受の基盤」と位置づけ、DHS・商務省・財務省・War 省・NSA・ホワイトハウスに 30〜60 日での実行を要求。

- 重要インフラの「ラストマイル問題」に対し、CISA 主管の Sensitive Remediations Program 創設を提言。リソース不足の水道事業者等への官民ハンズオン修復を想定。
- OSS の脆弱性はリポジトリを「戦略的執行点」とし、一定規模以上のプロジェクトをプロアクティブにスキャン・通知する opt-in 方式を提案。

出典: CrowdStrike Patch Tuesday / Security Affairs newsletter Round 582（2026-06-21）

出典: BleepingComputer（2026-06-21）

3. C2 喪失後の持続化 — Tailscale と OpenSSH

出典: Security Affairs newsletter Round 582（2026-06-21）

- フォレンジックでは未承認の Tailscale／SSH 構成・サービスの検出が侵害範囲特定の鍵。

4. LOTL 手法の普遍化（IR への含意）
