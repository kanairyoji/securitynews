地政学リスク・国家サイバーオペレーション

カテゴリ: カテゴリ1「地政学リスク・国家サイバーオペレーション」
対象日: 2026年06月27日
生成日時: 2026-06-28

総評

本日はロシア情報機関（RIS）による暗号メッセージアプリ（Signal等）を標的とした長期フィッシング作戦が最大のトピックとなった。FBI/CISA とウクライナ保安庁（SSU）が相次いで警告を発し、FSB 系の UNC5792・UNC4221 が攻撃手法を「認証コード窃取」から「Signal バックアップ復旧キー窃取」へと進化させたことが明らかになった。米国務省は UNC5792 に最大1,000万ドルの懸賞金を提示し、外交・法執行面でも対抗措置が強化されている。攻撃は政府・軍・ジャーナリスト・ウクライナ要人を狙った持続的かつ現在進行中の国家オペレーションである。

1. ロシア情報機関、Signal 復旧キー窃取へ手口を進化（FBI/CISA 更新警告）

出典: Security Affairs（2026-06-27）/ The Hacker News（2026-06-27）

  • FBI/CISA は2026年3月の警告を更新し、RIS が主目的を「SMS 認証コード窃取」から「Signal バックアップ復旧キー窃取」へ転換したと公表。
  • 追跡名は UNC5792・UNC4221。FSB 国境警備局所属の将校やロシア軍系のために活動する人物と関連。
  • 手口: Signal の「サポートボット」を装い、二要素認証導入やデータ復旧を口実にバックアップ有効化→復旧キーの貼り付けへ段階的に誘導。
  • 復旧キーは過去メッセージ全履歴とアカウント乗っ取りを可能にし、同一電話番号で再作成した新アカウントにも有効なまま残る点が深刻。
  • 重要な前提: Signal の暗号やアプリ自体は破られていない。正規機能を悪用した「アカウント＝弱点」型の攻撃。対策は設定から復旧キーを再生成すること。

2. ウクライナ SSU、ロシア情報機関の偽サポートSMS作戦を FBI と共同摘発

出典: The Hacker News（2026-06-27）

  • SSU は FBI と共同で、ウクライナ・欧州・米国の政府・軍・政治家・活動家のメッセージアカウントを狙う長期作戦を摘発。
  • 標的は組織・要人だけでなくウクライナ国民の個人アカウントにも拡大。軍事・政治・経済情報の窃取が目的。
  • 関連クラスタとして Star Blizzard、UNC5792（UAC-0195）、UNC4221（UAC-0185） を挙げる。
  • 別途、ベラルーシ系 UNC1151（Ghostwriter / UAC-0057） が侵害済みアカウントから情報窃取マルウェア OYSTERBLUES を配布するスピアフィッシングも CERT-UA が報告。

3. 米国務省 Rewards for Justice、UNC5792 に最大1,000万ドルの懸賞金

出典: Security Affairs（2026-06-27）

  • 助言発表に合わせ、UNC5792 に関する情報に最大1,000万ドルを提示。
  • 蘭情報機関、独 BfV/BSI、仏 ANSSI の今年の警告と重複。Google GTIG が2025年初頭に文書化した Signal 連携デバイス悪用の延長線上。同手口は WhatsApp・Telegram でも観測。

カテゴリ: カテゴリ2「国家サイバー戦略・法制度」
対象日: 2026年06月27日
生成日時: 2026-06-28

総評

米国の AI・サイバーセキュリティ政策が大きく動いている。2026年6月2日にトランプ大統領が署名した大統領令（EO）14409「先端AIイノベーション・セキュリティ促進」が、今後30〜60日で連邦政府・産業界・研究コミュニティに具体的アクションを求めている。これと連動し、OpenAI は攻撃的サイバー能力を持つ最新モデル GPT-5.6（Sol/Terra/Luna）を米政府との連携の下で限定プレビュー公開した。AI のサイバー能力を「covered frontier model（対象先端モデル）」として政府が評価・管理する枠組みづくりが進む。

1. 大統領令14409と「次の一手」（CrowdStrike の政策提言）

出典: CrowdStrike Blog（2026-06）

  • EO 14409 は「セキュリティが AI の便益享受の基盤」と位置づけ、DHS・商務省・財務省・War 省・NSA・ホワイトハウスが連携して施策を具体化。
  • 業界では AI が大量のバグを発見し手作業のレビュー・パッチを圧倒する「vulnpocalypse（脆弱性の黙示録）」が懸念され、60日という速度は妥当とされる。CrowdStrike 2026 脅威レポートでは AI 活用攻撃が前年比 89%増。
  • 提言: ①連邦政府への AI 検知・対応（AIDR）ツール導入（EO14028 の EDR 義務化に倣う）②アイデンティティ保護の強化 ③エージェント型 SOC の活用 ④重要インフラへの「機微な修復プログラム」設置 ⑤OSS リポジトリを戦略的な脆弱性発見の起点とする。

2. OpenAI、GPT-5.6 を米政府連携の下で限定プレビュー — 「covered frontier model」枠組み

出典: The Hacker News（2026-06-27）

  • OpenAI は GPT-5.6（フラッグシップ Sol、バランス型 Terra、高速・低コスト Luna）を少数企業に限定公開。米政府承認済みの信頼パートナーのみ。
  • トランプ大統領は今月、AI モデルの能力を連邦政府が評価し「covered frontier models（先端サイバー能力を持つ AI）」を指定する枠組みを求める EO に署名。
  • Anthropic も Mythos AI を一度市場撤回後、重要インフラを防衛する約100の信頼企業・連邦機関への提供を政府が許可。Fable 5 の一般提供再開も政府と調整中。

1. NAIC、サイバー攻撃で投資リスク格付業務を一時停止

出典: DataBreaches.net（2026-06-27）

  • 全米保険監督官協会（NAIC）は米50州・DC・5準州の保険監督官で構成される標準設定・規制支援組織。
  • サイバー攻撃を受け、投資リスク格付（investment risk designations）業務を一時停止。保険・金融セクターの規制機能に直接影響。

2. EO 14409、重要インフラ防衛の「ラストワンマイル」問題に対応

出典: CrowdStrike Blog（2026-06）

  • Sec.2(c)(i): CISA 権限を活用し連邦民生システムのサイバー防衛を優先・加速（最も緊急のタスク）。
  • Sec.2(c)(iii): 地方水道事業者などリソース不足セクター向けに、CISA が運営し民間と連携する「機微な修復プログラム（Sensitive Remediations Program）」設置を提案。研究者が AI で脆弱性を発見しても修復能力がない問題に対処。
  • CrowdStrike の Frontier AI Readiness and Resilience Service / Project QuiltWorks が即応可能と提示。

3. Anthropic Mythos、重要インフラ防衛者へ再提供

出典: The Hacker News（2026-06-27）

  • 米政府は Anthropic に対し、重要インフラを運用・防衛する約100の信頼企業・連邦機関への Mythos AI 提供を許可。サイバー特化モデルへのアクセスを迅速に復旧中。

1. 攻撃的サイバー能力を持つ AI モデルへのアクセス統制

出典: The Hacker News（2026-06-27）

  • OpenAI は GPT-5.6 Sol を「これまでで最も能力の高いサイバー向けモデル」と位置づけ、脆弱性研究・エクスプロイト開発により適合すると説明。ExploitBench で Anthropic Mythos Preview に匹敵する性能を約1/3の出力トークンで実現。
  • 一方で、自律的なエンドツーエンド攻撃や脆弱性の実戦兵器化までは到達していないとし、防御者（コードレビュー・脆弱性研究・パッチ開発・防御テスト）への提供に限定。
  • 米政府は AI モデルの能力を評価し「covered frontier models（先端サイバー能力を持つ AI システム）」を指定する枠組みを EO で要求。デュアルユース技術の流通管理がモデル単位で行われる構図。

2. AI モデルの市場アクセス管理（Anthropic Mythos の撤回と再提供）

総評

本日の DFIR 観点では、TonRAT キャンペーンの永続化機構が「封じ込めの落とし穴」を示す好例として注目される。単一の永続化パスをブロックしても二次パスが生き残り再活性化するため、複数機構を同時に除去する完全修復が必須となる。また DirtyClone はディスクに痕跡を残さずオンディスク完全性監視を回避するため、フォレンジック手法の前提を揺るがす。ClickOnce 悪用も正規プロセスツリー内で実行されるため、検知・ハンティング戦略の見直しが求められる。

1. TonRAT — 二重永続化と「ブロックしても生き残る」復旧パス

出典: Security Affairs / Microsoft Threat Intelligence（2026-06-27）

  • 永続化は HKCU\Run（Node.js 起動点） と HKCU\RunOnce（ProgramData ペイロードを毎回再生成するループ） の二重構成。
  • 実例: Defender が PE ペイロード xmnrwv9l.exe をブロックしたが Node.js Run キーは残存し、2日後に新 C2 へ再接続・再活性化。
  • 完全修復には両機構を同時除去: RunOnce エントリ、Node.js を指す Run キー、Node.js ランタイム本体、AppData\Local\Nodejs\ 配下の全 .js ファイル。
  • ハンティングピボット: RunOnce ループ、非標準ポート（56001-56003, 8443/8445/8453, 5555）への C2 ビーコン、ユーザー空間に Node.js が出現する端末。

2. DirtyClone — 痕跡を残さない LPE がフォレンジックを回避

出典: Security Affairs / JFrog（2026-06-25/27）

  • メモリ上で実行ファイルを書き換えるためカーネルログ・監査証跡を残さず、オンディスク完全性監視を回避。ディスク上のファイルは無傷のまま su が root を渡す。
  • 検知が困難なため、エクスプロイト前提条件（CAP_NET_ADMIN 取得、unshare -Urn による名前空間作成、esp4/esp6/rxrpc モジュール）の監視が有効。

3. ClickOnce 悪用の検知戦略

出典: CrowdStrike Blog（2026-06）

  • 悪意あるペイロードが rundll32.exe・dfsvc.exe の正規プロセスツリー内で実行されステルス性が高い。.appref-ms のスタートアップ配置やスケジュールタスク悪用が永続化の指標。Falcon センサーによる検知・対応をデモ。
