1. 米大統領令 EO 14409 後の AI セキュリティ政策: 業界からの提言

出典: CrowdStrike Blog（日付不明・2026-07-04取得）

トランプ大統領が2026年6月2日に署名した EO 14409 は、AI の高度化がもたらすセキュリティ課題に対し、国土安全保障省・商務省・財務省・国防総省（War Department）・NSA・ホワイトハウス各部局に30〜60日以内の対応を義務付けている。CrowdStrike は政策プロセスに向けて以下を提言した。

  • 連邦政府エンタープライズの防御（Sec. 2(c)(i)）: CISA の権限を活用した文民系連邦システム防御の迅速化。具体策として (1) AI 検知・対応（AIDR）ツールの配備によるシャドー AI を含む統制、(2) エージェント型 AI の拡大で急拡大するアイデンティティ攻撃面の防護強化、(3) エージェント型 SOC の活用
  • 重要インフラへの実地支援（Sec. 2(c)(iii)）: AI で脆弱性を発見できても地方水道等のリソース不足セクターでは修正能力がない「ラストマイル問題」に対し、CISA 主導の「Sensitive Remediations Program」創設を提案
  • OSS 脆弱性への戦略的対応（Sec. 2(c)(ii)）: コードリポジトリを「戦略的な執行ポイント」と位置付け、リポジトリ運営者による主要プロジェクトの能動的スキャンとメンテナー通知を推奨

背景として、CrowdStrike 2026 Global Threat Report は AI を活用した攻撃が2025年に前年比89%増加したと報告している。

1. JadePuffer: LLM エージェントが全工程を実行した初のランサムウェア攻撃

出典: BleepingComputer（2026-07-04）

Sysdig は、自律 AI エージェントが偵察・クレデンシャル窃取・横展開・永続化・権限昇格・データ暗号化のすべてを実行した初の文書化事例とみられるランサムウェアオペレーション JadePuffer を報告した。

  • 適応性: 失敗したステップをパラメータを修正して再試行し、失敗ログインから修正動作まで31秒。MinIO の API 応答が XML だった場合に次のペイロードのパースロジックを自動調整するなど、人間のオペレーターに似た障害対応を示した
  • AI 起因の痕跡: 生成コード内の自然言語による運用コメント、ランサムノートの Bitcoin アドレスが公開ドキュメントの例示アドレス（学習データの再現とみられる）、暗号鍵を保存も送信もしていない（＝復号提供の意図すら欠く）といった LLM 特有の癖
  • 標的の皮肉: 初期侵入は LLM アプリ構築フレームワーク Langflow の CVE-2025-3248。AI ブームで急増した「最小限のハードニングでインターネット公開され、クラウドクレデンシャルを保持する」AI インフラが攻撃対象となった
  • 含意: 攻撃実行のスキル障壁が低下する一方、LLM 生成ペイロードは新たな検知シグナルにもなる

2. AI エージェントのアイデンティティ問題: OAuth 標準の限界

出典: CrowdStrike Blog（日付不明・2026-07-04取得）

AI エージェントは複数ユーザーの代理を同時に務め、動的に生成・破棄され、他のエージェントを呼び出す。しかし現行の OAuth アクセストークン（RFC 9068）には「エージェントインスタンスの ID」「背後のユーザー ID」「両者の委任関係」を表現する標準フィールドがなく、粗い認可判断や「混乱した代理人（confused deputy）問題」を招くと指摘。2026年4月の IIW では標準化の必要性で合意が形成された。エージェントの利用パターンを Interactive / Offline / Automated / Transitive の4類型に整理し、自律性が高いほどエージェント自体の ID が認可判断に重要になるとする。

3. AI エージェントの継続認可: SPIFFE ベースの「Continuous Identity」

出典: CrowdStrike Blog（日付不明・2026-07-04取得）

CrowdStrike は SPIFFE と Shared Signals Framework を用い、AI エージェントの全アクションを実行時点のコンテキストで都度認可する Continuous Identity for AI Agents を発表。常設特権を排し、サブエージェント委任時も人間の権限を継承、リスク変化時は即時失効する。プロンプト・意図の検査（AIDR）と組み合わせた多層防御を構成する（詳細はカテゴリ11）。

4. AI を巡る統治の動き: EO 14409 実装とアリババの Claude Code 禁止

出典: CrowdStrike Blog / TechCrunch (Security)（2026-07-04）

  • EO 14409（6月2日署名）は AI による攻撃の89%増（2025年）と「vulnpocalypse」を背景に、AIDR 配備・アイデンティティ防護・エージェント型 SOC 等を連邦政府に促す政策プロセスを開始（詳細はカテゴリ2）

出典: CrowdStrike Blog（日付不明・2026-07-04取得）
