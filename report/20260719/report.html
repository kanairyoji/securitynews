1. ロシア政府機関を狙う「HelloNet」— 国産セキュリティ製品ViPNetの更新機構を悪用

出典: BleepingComputer（2026-07-19）

  • 2026年5月から活動。悪性DLL（wtsapi32.dll＝HelloInjector）を正規の itcsrvup64.exe 経由でサイドロードし、svchost.exe へ注入
  • 展開されるツール群: HelloProxy・バックドア HelloExecutor・ログ消去用 HelloCleaner・Rust製 HelloBackdoor
  • sina.com を参照する文字列や中国科学技術大学のミラー利用から中国語話者APTに低確度で暫定帰属（偽旗の可能性も排除せず）
  • 監視推奨ポート: 5003 / 5060 / 443

ロシアの「信頼された国産セキュリティ基盤」自体が侵入経路となった点で、露中関係の水面下の緊張を示唆する事案として追跡価値が高い。

出典: SANS Internet Storm Center（2026-07-19）

SANS ISC のハニーポットが、Hikvision 製監視カメラの REST API「ISAPI（Intelligent Security API）」のエンドポイント /ISAPI/System/status を狙う新たな偵察スキャンを検知した。

  • 同APIはカメラ設定の完全制御が可能で、Basic/Digest 認証へのパスワードブルートフォースの足がかりになり得る
  • AES-CBC による暗号化も Basic 認証併用時は実質的に意味をなさない
  • Hikvision カメラは長年脆弱性が多発しボットネットの標的となってきた経緯があり、カメラ類をインターネットに直接露出させないよう改めて警告

監視カメラは電力・交通・庁舎等の重要施設に広く設置されており、物理セキュリティ機器のネットワーク分離は重要インフラ防護の基本動作として徹底したい。

  • 0-RTT リプレイ攻撃対策（Early-Data ヘッダ検証）、ML による Website Fingerprinting（精度90%超）

QUIC 対応を「単なるプロトコル追加」ではなく検知アーキテクチャの再設計として扱うべきことを示す良質な技術解説である。
