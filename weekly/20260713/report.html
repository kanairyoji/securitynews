今週のエグゼクティブサマリー

今週は、「社外との接点になる基盤製品」への攻撃が一斉に火を噴き、その火の粉が国内の事業停止として実際に着地した週だった。

第一に、多くの企業が使う文書共有・グループウェア基盤の欠陥が週を通じて次々と悪用され、米当局は週内に関連する欠陥を計4件、悪用済みリストに追加して連邦機関に3日以内の修正を命じた。しかもこの製品には「修正の後半が来月まで出ない」欠陥の連鎖が残っており、パッチを当てても攻撃経路が完全には閉じない状態のまま翌週に持ち越されている。あわせて、リモートアクセス用の境界装置やセキュリティ検査アプライアンスでも、最大深刻度の欠陥がゼロデイとして悪用されていたことが判明した。境界装置は「パッチを当てて終わり」ではなく、「すでに侵入されていた前提で調査する」対象になった。

第二に、国内企業への攻撃被害が今週は例外的に集中した。大手低温物流グループが攻撃で冷蔵倉庫の入出庫・冷凍食品の出荷を止め、外食・小売に波及したまま週後半にようやく段階復旧に入った。国内最大手のタクシー事業者も配車システムを止め、化学メーカー・車両機器メーカーの被害も明らかになった。先週の海外事例（攻撃後の破産）に続き、「サイバー攻撃＝物流・生産ラインの停止」が国内でも現実になった。

第三に、欧州と英国が初めて足並みを揃え、ロシア情報機関のサイバー部門に対する制裁・攻撃の公式帰属・13カ国共同の技術勧告を同日に発動した。名指しされた攻撃手口は「10年前から知られた欠陥と弱い設定の放置」であり、地政学リスクがそのまま自社インフラの設定点検項目に変換される時代に入った。

第四に、AIを使った攻撃が「研究」から「実務」に移った。AIに攻撃作業の9割を任せた単独犯や、AIが約40分で企業ネットワークの管理者権限まで到達する実証が報告される一方、企業が導入するAIエージェント製品側には未修正の欠陥が複数残り、AIエージェント関連の侵害を経験した組織は9割に達するという調査も出た。あわせて、ランサムウェア侵入の約8割が「盗まれたパスワード」起点（脆弱性悪用は2割弱に低下）という年次データが示され、投資の重心をID防御に移す根拠が揃った。

CISO視点: 今週の経営リスク

# リスク 事象 経営としての判断事項
1 国内でも「攻撃＝事業停止」が現実化 国内の物流・交通・製造の複数社が同週に被害を公表。冷蔵物流の停止は取引先の販売機会損失まで波及し、復旧に週をまたいだ 自社の停止時損失の定量化（1日あたり）と、手作業での代替運用をどこまで維持できるかの検証。BCP演習に「基幹システム1週間停止」シナリオを含める
2 境界・基盤製品は「侵入済み前提」で扱う 境界装置のゼロデイ悪用は修正公開の3週間以上前から始まっていた。文書共有基盤は修正の一部が来月まで出ない パッチ適用に加えて侵害調査（認証情報・セッションの窃取確認）までをワンセットとする方針の承認。調査費用の事前予算化
3 ID起点の侵入が支配的に ランサムウェア侵入の約8割が窃取された認証情報起点。大手医療企業への恐喝も「ヘルプデスクへの電話」からIDセンターの突破で成立 投資の重心を脆弱性対応一辺倒からID防御（MFA強制・ID脅威検知・ヘルプデスクのコールバック義務化）へ再配分するか
4 AIエージェントの導入statとリスクが同時に露呈 AIエージェント関連の侵害経験は組織の9割との調査。ブラウザ操作型AI製品には未修正欠陥が残り、開発者向けAIツールの設定ファイルが攻撃者の収集対象になった 未修正欠陥が公表されているAIエージェント製品の一時利用停止を判断できる体制と、エージェントの権限・ID分離を導入の前提条件にするか
5 制裁・規制対応が同時多発 欧英の対露制裁で大手SNS・国営メッセンジャーが制裁対象化。防衛取引の認証制度は米国で凍結・見直し。遺伝子検査企業は州との和解で累計1億ドル規模の支払いに 制裁スクリーニングの更新、規制が緩んでも自主水準を維持する方針の明文化、MFA等の基本統制不備が賠償に直結する前提での投資判断
6 サードパーティ経由の侵害が引き続き主経路 大手会計事務所は委託先のIT管理ツール経由で税務文書が流出。AI音楽サービスも従業員認証情報の窃取（供給網攻撃起点）から侵害された 委託先のID管理・アクセス経路を自社の攻撃面として棚卸しする。契約への侵害時対応条項の織り込み

CTO視点: 技術潮流と中長期の備え

# 潮流 今週の裏付け 中長期の備え
1 「信頼の仕組み」そのものが攻撃対象に コード署名証明書の発行元侵害（証明書60枚失効）、正規の来歴証明付きで配布されたマルウェア入りパッケージ、国産セキュリティ製品の更新機構悪用、起動時の署名検証の欠陥が同一週に集中 署名・来歴・更新チャネルを「信頼して終わり」にせず、検証を多層化する設計へ。証明書失効・パッケージ改ざんの監視を供給網管理に組み込む
2 パッチ運用は「量×速度」の二重苦へ 月例修正が過去最多の約620件（AIによる脆弱性発見が押し上げ）。悪用はパッチ公開の平均7日前に始まり、修正公開の数時間後に別の未修正欠陥が公開される事例も 全量適用主義を捨て、悪用兆候ベースの優先順位付け＋検知・封じ込め時間の短縮へ。修正キャパシティを事業制約として定量化
3 AIエージェントのID・権限が業界標準の空白のまま ローカルAIエージェントの2割超が人間の資格情報を借用して本番データへアクセスし、監査ログで区別不能との調査。ID統合管理の新興企業に大型投資が集中 エージェント単位の一意ID・代理関係の明示・長期クレデンシャル禁止を自社標準として先に定義（先週からの継続課題。標準の空白は当面続く前提で）
4 攻撃の自動化は「モデルよりハーネス」 汎用AIに攻撃用の足回りを与えるだけで侵入からドメイン掌握まで自律実行できることが実証され、同等能力が1年以内にオープンソース化するとの警告 侵入後の横展開を止める内部分割・ID駆動の最小権限（先週の「内側がフラット」問題への対処）を、AI攻撃の速度を前提に前倒し
5 EoL資産が「13年潜伏」の温床に サポート終了の開発基盤経由で13年間未検知だった国家系バックドアが発見された。旧OSの残存端末は現行版の3倍の欠陥を抱えるとの調査。産業機器では販売終了品の修正なし事例も EoL資産の棚卸しを「いつか」から期限付き計画へ。除去できない資産はネットワーク分離を唯一の防御策として設計
6 脆弱性対応の官民インフラが再編へ 米政府がAI活用の脆弱性クリアリングハウス構想を始動。国内では重要インフラ向け「パッチ適用の外部サービス化」も発表された 自社の脆弱性運用をこれら外部パイプラインと接続できるよう、資産・SBOM・優先順位付けのデータ整備を先行

推奨アクション（経営判断が必要なもの）

1. 外部公開している基盤製品の「パッチ＋侵害調査」を今週分は特別扱いで完遂させる

文書共有基盤・境界装置・セキュリティアプライアンスの今週の欠陥群は、修正公開前から悪用されていたものを含む。適用済みかどうかではなく「侵入されていないか」までの確認を、費用を付けて完了させる（対象と手順は実務者レポート側に整理済み）。

2. 「基幹システム1週間停止」を想定した事業影響の定量化とBCP演習を指示する

国内の同業・近隣業種で物流・生産停止が現実に起きた。停止時損失（1日あたり）・手作業代替の限界・復旧優先順位を数字で持ち、取引先への影響通知の基準と体制まで含めて演習する。

3. セキュリティ投資の重心をID防御へ再配分する方針を決める

侵入の約8割がID起点というデータが出た。MFAの例外撲滅、ヘルプデスクのコールバック義務化（先週からの継続）、ID脅威検知の導入、そして2027年の旧式MFA廃止期限に向けたパスキー移行計画を、一つのプログラムとして予算化する。

4. AIエージェント製品の利用統制ルールを今期中に確定する

未修正欠陥が公表されている製品の一時停止判断、エージェントへの権限付与の上限、開発者向けAIツールの設定ファイル・認証情報の保護を含む。「便利だから黙認」の状態は、9割の組織がエージェント関連侵害を経験しているという調査結果と両立しない。

5. 制裁・規制変化への対応を1件ずつ棚卸しする

欧英の対露制裁対象（SNS・メッセンジャー等）のスクリーニング反映、米防衛認証制度の凍結を受けても自主水準を維持する方針の明文化、モバイルOSのAI開放命令（欧州）を踏まえた社用端末統制の見直し。いずれも実務は軽いが、方針決定は経営マターである。

総評

今週の主戦場は「外部公開された基盤製品」だった。中心は Microsoft SharePoint で、週を通じて 4件が KEV 入り（CVE-2026-32201 / 45659 / 56164 / 58644）し、うち CVE-2026-56164 は Patch Tuesday の悪用確認済みゼロデイ、CVE-2026-58644 は CVSS 9.8 の未認証RCE。さらに JWT 認証バイパス CVE-2026-55040 は連鎖先の RCE 側修正が8月予定のため、パッチを当てても攻撃面が完全には閉じない「未完の連鎖」を翌週に持ち越す。2025年の ToolShell 大量悪用の再来が現実的な懸念になっている。

境界機器では SonicWall SMA1000 のゼロデイ2件（CVE-2026-15409 CVSS 10.0 / CVE-2026-15410）が Inc ランサムウェア関与のアクター（UTA0533）に 6/22 から悪用されていたことが判明。修正公開（7/14）の3週間以上前から侵入されていた計算で、「パッチ適用＝安全」ではなく侵害調査までがワンセットである。FortiSandbox の CVSS 9.1×2件（KEV、連邦期限 7/19）も同型。

前週最大の持ち越しだった Progress ShareFile は「パストラバーサルのゼロデイ悪用」と確定し、修正版が出て一応の決着を見た（CVE番号は今週後半〜来週公表見込み）。一方、新たな未解決として、研究者 Nightmare Eclipse（Chaotic Eclipse）が Patch Tuesday 数時間後に Windows の未修正ゼロデイ「LegacyHive」の PoC を公開（CVE未採番・パッチ未提供）。同氏の連続公開は9件目で、過去分（BlueHammer 等）はランサムウェアに悪用された実績があり、短期武器化が警告されている。

量的にも Microsoft の月例修正が過去最多の約620件（2カ月連続記録更新、AI による脆弱性発見が押し上げ）、Mandiant は平均悪用時間がマイナス7日（パッチ公開前に悪用開始）と報告。全量適用は破綻しており、KEV・悪用兆候ベースの優先順位付けが前提になる。

もう一つの週貫通テーマは国内企業への攻撃集中である。ニチレイ（7/13 攻撃確認、冷蔵倉庫・出荷停止、7/17 段階復旧開始）、日本交通（配車システム停止、AiLock 疑い・リークサイト掲載）、日産化学（7/1 検知、調査中）、極東開発工業（Inc Ransom 掲載）、The Gentlemen による国内2社掲載（カネコ・ディンク）と1週間で並んだ。TTP 面では、Sophos 年次の「ランサム初期侵入の79%が窃取認証情報起点」（脆弱性悪用は18%に低下）が今週最重要の統計で、ID 防御・ITDR への投資シフトを裏付ける。

7. インシデントレスポンス・フォレンジック（DFIR）

  • SonicWall SMA1000 侵害調査の起点: /wsproxy アクセスログ、プロセス注入痕跡、nginx 設定改変、setuid バイナリ確認（Volexity IoC 参照）。
  • TfL ポストモーテム: 148システム停止・2.7万人の対面パスワードリセット・復旧費2,900万ポンド。侵入はヘルプデスクへの 2FA リセット要求の反復。ヘルプデスク手順が IR コストを桁で左右する実例。
  • ACR Stealer（Microsoft 解析）: ClickFix → mshta.exe → JPEG 内ペイロードのファイルレス実行。対応はセッショントークン失効を最優先。
  • クラウド IR: NadMesh 感染時はクラウドキーを「ローテーションでなく失効」、SSH authorized_keys＋cron の三重永続化除去。Rapid7 が AWS 永続化（iam:CreateUser、AssumeRole バックドア、Lambda）の CloudTrail 検知手順を公開。
  • 13年潜伏の教訓: 台湾ハイテク製造で中国系ルートキット Daxin と新バックドア Stupig（kbdus1.dll 偽装、winlogon 経由 SYSTEM 実行、2013年コンパイル）を発見。侵入経路は EoL の JDK 1.5/1.6 を使う旧版 SSO ポータル。winlogon への不審 DLL 読み込み監視と EoL ミドルウェア棚卸しが検知の入口。

8. 脅威インテリジェンス・マルウェア解析

  • ClickFix（偽CAPTCHA）が犯罪・国家系の双方で主流化: GRU 系 Sandworm がウクライナで10超の改ざんサイトから展開（GhettoVibe / ScoutCurl / FluidLeech / LoadLoop）、macOS では ClickLock（アプリを210ミリ秒間隔で最大83時間強制終了しパスワード入力を「強要」、33カ国100人超）。
  • 国家系の新規作戦: 露国産 VPN 製品 ViPNet の更新機構を悪用する「HelloNet」（露標的、低確度で中国語話者に帰属）、東南アジア政府標的の Go 製「GoSerpent」、DigiCert 侵害が中国系 GoldenEyeDog 配下「CylindricalCanine」と特定（EVコード署名証明書60枚失効、Zhong Stealer 署名に悪用）。北朝鮮系 Contagious Interview は偽コーディング試験で SVG ステガノグラフィ配布。
  • ID 攻撃の産業化: OAuth クライアントID偽装・Forg365・Jalisco / OmegaLord（継続ウォッチ参照）。テキストソルティングで LLM メールフィルタを回避する試行が100万件超（Barracuda）。
  • macOS スティーラー多様化: CrashStealer（正規 Apple Developer ID 署名、約80ウォレット拡張・14パスワードマネージャー標的）、OkoBot（ハードウェアウォレットの偽リカバリー画面でシード窃取、25カ国）。Telegram セッションファイル複製による 2FA 回避乗っ取りも。
  • LLM 製ボットネット: TuxBot v3（17アーキテクチャ対応、25超の既知CVE収録、機能の約30%が LLM 由来バグで不動作）——「未完成でも量産される」段階に入った。

9. 脆弱性・ライフサイクル管理

週内出現 CVE は80件（前週115件）。最上位は要対応リストのとおり SharePoint 4件 KEV・SonicWall・FortiSandbox。構造面では、①Microsoft 月例が約620件と2カ月連続で過去最多（AI による発見「Multi-Model Agentic Scanning Harness」が主因）、②Mandiant の平均悪用時間マイナス7日、③修正数時間後の未修正ゼロデイ PoC 公開（LegacyHive）、と「量の爆発 × 悪用の前倒し × 非協調公開」の三重苦が確定した。CISA・NSA 等は協調的脆弱性開示（CVD）プログラム構築の国際ガイダンスを公開し、米政府は AI 活用の脆弱性クリアリングハウス「Gold Eagle」を始動。実務は「全部当てる」から KEV・悪用兆候・露出資産ベースの優先順位付け＋期限管理への移行を迫られている。なお Rapid7 は AttackerKB 公開版を8月18日で終了予定。

10. ソフトウェアサプライチェーン・OSSセキュリティ

  • AsyncAPI npm 攻撃（週200万DL超）: GitHub Actions CI/CD 侵害により正規の SLSA 来歴証明付きでマルウェアを約4時間配布（7/14 07:10〜11:18 UTC）。約9.2万行のフレームワークが IPFS・BitTorrent・Ethereum スマートコントラクトをフォールバック C2 とし、npm/PyPI/Cargo トークン発見時に自己増殖。来歴証明の検証だけでは防げない事例。ロックファイル再生成と認証情報ローテーションを。
  • その他の npm/NuGet: Jscrambler がメンテナー認証情報侵害で悪性化（7/13）、Injective Labs SDK 侵害、@vitejs 偽装「ViteVenom」7件（北朝鮮系 PolinRider 関連指摘）、悪性 NuGet 11件。npm 12 はインストールスクリプト既定無効化・8月からトークン制限・2027年1月に直接公開廃止と、CI/CD 側の対応期限が具体化。
  • 信頼基盤への攻撃の同時多発: DigiCert 証明書失効60枚、ViPNet 更新機構悪用（HelloNet）、UEFI shim の Secure Boot バイパス、U-Boot FIT 署名検証欠陥6件——「署名・更新・起動」の信頼チェーン全層で欠陥・悪用が出た週。
  • 開発者端末が下流侵害の入口に: Contagious Interview が .claude / .cursor / .gemini 等の AI ツール設定を収集。Grok Build がリポジトリ・SSH 鍵を無断送信していた問題も。開発端末の秘密情報分離を。

11. 防御アーキテクチャ・セキュリティ製品

※ベンダー発信・自社調査の数値は参考値として扱うこと。

「アイデンティティを制御平面に」が今週の主調。Token Security はローカル AI エージェントの2割超が人間の資格情報を借用して本番データへアクセスし監査ログで区別不能と報告し、人間・AI・マシン ID 統合管理の Oak が6,000万ドルのシードを調達した。防御側の AI 化も加速: Google Cloud の「エージェンティック・ディフェンス」戦略（Wiz 統合）、ソフトバンク×OpenAI の重要インフラ向け「Patching as a Service」、Microsoft の Defender Experts のマルチクラウド拡張。一方で Defender 自体の欠陥（RoguePlanet パッチ疑義）や Bind Link による EDR 回避が出ており、「防御製品を無条件に信頼しない」検証設計（改ざん防止・ブロックモード・回避手法のハンティング）が引き続き必要。SASE には TLS 1.3・証明書ピンニングで検査不能になる「AI 盲点」の指摘。Cloudflare の WordPress 向け WAF 即日全プラン展開は、CDN/WAF 層の緊急対応が実質的な業界インフラになりつつあることを示した。

13. 先端技術・新領域セキュリティ

  • Agent Data Injection（ADI）: Claude in Chrome / Claude Code / Codex / Gemini CLI 等の実製品で成功率31〜43%、修正未提供。前週の MemGhost（永続メモリへの虚偽記憶注入）に続き、エージェントの「記憶・データ・命令の未分離」が攻撃クラスとして確立。
  • 未修正のエージェント製品欠陥: Claude for Chrome の合成クリック（Event.isTrusted 未検証、5/21 報告後も未修正）、Cursor IDE（2025年12月報告後未修正）、Android 16 の Gemini 経由ロック画面バイパス（修正展開中）。「AI エージェント製品は未修正欠陥を抱えたまま流通している」前提で利用統制を。
  • 攻撃自動化: Cato の GPT-5.5 実証（40分で AD 掌握、「モデルよりハーネス」）、Intruder の自動脆弱性発見パイプライン（WordPress プラグインの SQLi CVE-2026-3985 を自動発見）——攻守双方で自動化が実装段階。
  • 標準化の動き: AI エージェント決済の x402 Foundation 発足（Linux Foundation 傘下40社）、Ledger の Agent Stack（機微操作はハードウェア署名必須）、Mandiant の AI 開発パイプライン堅牢化指針（MCP プラグイン汚染・間接プロンプトインジェクション・短命 JIT トークン）。
