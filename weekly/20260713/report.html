今週のエグゼクティブサマリー

今週は、「社外との接点になる基盤製品」への攻撃が一斉に火を噴き、その火の粉が国内の事業停止として実際に着地した週だった。

第一に、多くの企業が使う文書共有・グループウェア基盤の欠陥が週を通じて次々と悪用され、米当局は週内に関連する欠陥を計4件、悪用済みリストに追加して連邦機関に3日以内の修正を命じた。しかもこの製品には「修正の後半が来月まで出ない」欠陥の連鎖が残っており、パッチを当てても攻撃経路が完全には閉じない状態のまま翌週に持ち越されている。あわせて、リモートアクセス用の境界装置ではゼロデイ悪用が修正公開の3週間以上前から始まっていたことが判明し、セキュリティ検査用アプライアンスでは欠陥の公表から24時間以内の悪用が確認された（当局の修正期限は週末に到来）。週末には、世界のWebサイトの大半が使うサイト構築基盤でも修正公開の翌日に攻撃手順が公開され、実際の悪用が始まった。境界装置・公開サーバーは「パッチを当てて終わり」ではなく、「すでに侵入されていた前提で調査する」対象になった。

第二に、国内企業への攻撃被害が今週は例外的に集中した。大手低温物流グループが攻撃で冷蔵倉庫の入出庫・冷凍食品の出荷を止め、外食・小売に波及したまま週後半にようやく段階復旧に入った。国内最大手のタクシー事業者も配車システムを止め、化学メーカー・車両機器メーカー・建設ソフト・産業機器の被害も明らかになった。さらに週末には宅配大手で（攻撃ではなく）システム復旧作業の設定誤りにより約7万人規模の個人情報誤表示が公表され、攻撃起因でなくても通知・公表対応は同じ重さで走ることを示した。海外では犯行グループが国営石油会社を恐喝し「株価下落」を交渉圧力として明示するなど、上場企業の市場価値そのものを人質に取る手口も現れた。

CISO視点: 今週の経営リスク

# リスク 事象 経営としての判断事項
1 国内でも「攻撃＝事業停止」が現実化 国内の物流・交通・製造の複数社が同週に被害を公表。冷蔵物流の停止は取引先の販売機会損失まで波及し、復旧に週をまたいだ。海外では国営石油会社への恐喝で犯行グループが「株価下落」を圧力として明示 自社の停止時損失の定量化（1日あたり）と、手作業での代替運用をどこまで維持できるかの検証。BCP演習に「基幹システム1週間停止」シナリオを含める。上場企業は開示・IR対応も演習範囲に
2 境界・基盤製品は「侵入済み前提」で扱う 境界装置のゼロデイ悪用は修正公開の3週間以上前から始まっていた。検査アプライアンスは公表から24時間以内に悪用。文書共有基盤は修正の一部が来月まで出ない パッチ適用に加えて侵害調査（認証情報・セッションの窃取確認）までをワンセットとする方針の承認。調査費用の事前予算化
3 ID起点の侵入が支配的に ランサムウェア侵入の約8割が窃取された認証情報起点。大手医療企業への恐喝も「ヘルプデスクへの電話」からIDセンターの突破で成立（恐喝期限は週明け） 投資の重心を脆弱性対応一辺倒からID防御（MFA強制・ID脅威検知・ヘルプデスクのコールバック義務化）へ再配分するか
4 AIエージェントの導入拡大とリスクが同時に露呈 AIエージェント関連の侵害経験は組織の9割との調査。従業員の45%が無断AI利用、63%の組織はAIポリシー自体を欠く。ブラウザ操作型AI製品には未修正欠陥が残り、開発者向けAIツールの設定ファイルが攻撃者の収集対象になった 未修正欠陥が公表されているAIエージェント製品の一時利用停止を判断できる体制と、エージェントの権限・ID分離を導入の前提条件にするか。「証明可能なガバナンス」（非承認利用率等の自動計測）への移行
5 制裁・規制対応が同時多発 欧英の対露制裁で大手SNS・国営メッセンジャーが制裁対象化。防衛取引の認証制度は米国で凍結・見直し。遺伝子検査企業は43州連合と1,800万ドルで和解（累計1億ドル規模の支払いに）。年齢確認法制は世界30超の法域で施行段階 制裁スクリーニングの更新、規制が緩んでも自主水準を維持する方針の明文化、MFA等の基本統制不備が賠償に直結する前提での投資判断
6 サードパーティ経由の侵害が引き続き主経路 大手会計事務所は委託先のIT管理ツール経由で税務文書が流出。AI音楽サービスも従業員認証情報の窃取（供給網攻撃起点）から侵害された。セキュリティ事業者・MSSP自身の被害掲載も複数 委託先のID管理・アクセス経路を自社の攻撃面として棚卸しする。契約への侵害時対応条項の織り込み

CTO視点: 技術潮流と中長期の備え

# 潮流 今週の裏付け 中長期の備え
1 「信頼の仕組み」そのものが攻撃対象に コード署名証明書の発行元侵害（証明書60枚失効）、正規の来歴証明付きで配布されたマルウェア入りパッケージ、国産セキュリティ製品の更新機構悪用、起動時の署名検証の欠陥、CVE番号なしの「サイレント修正」が同一週に集中 署名・来歴・更新チャネルを「信頼して終わり」にせず、検証を多層化する設計へ。証明書失効・パッケージ改ざんの監視を供給網管理に組み込む。CVEに載らない修正も拾える棚卸し体制
2 パッチ運用は「量×速度」の二重苦へ 月例修正が過去最多の約620件（AIによる脆弱性発見が押し上げ）。悪用はパッチ公開の平均7日前に始まり、修正の数時間後に別の未修正欠陥が公開され、修正翌日に攻撃手順が公開されて翌々日に悪用が始まる事例まで出た 全量適用主義を捨て、悪用兆候ベースの優先順位付け＋検知・封じ込め時間の短縮へ。修正キャパシティを事業制約として定量化
3 AIエージェントのID・権限が業界標準の空白のまま ローカルAIエージェントの2割超が人間の資格情報を借用して本番データへアクセスし、監査ログで区別不能との調査。エージェント起点の検知が人間起点の2.5倍という観測も。ID統合管理の新興企業に大型投資が集中し、「AI検知対応（AIDR）」という新製品カテゴリも提唱された エージェント単位の一意ID・代理関係の明示・長期クレデンシャル禁止を自社標準として先に定義（先週からの継続課題。標準の空白は当面続く前提で）
4 攻撃の自動化は「モデルよりハーネス」 汎用AIに攻撃用の足回りを与えるだけで侵入からドメイン掌握まで自律実行できることが実証され、同等能力が1年以内にオープンソース化するとの警告。プロンプトインジェクションの技法分類は200超に拡大 侵入後の横展開を止める内部分割・ID駆動の最小権限（先週の「内側がフラット」問題への対処）を、AI攻撃の速度を前提に前倒し
5 EoL資産が「13年潜伏」の温床に サポート終了の開発基盤経由で13年間未検知だった国家系バックドアの詳細が週末に公表された。旧OSの残存端末は現行版の3倍の欠陥を抱えるとの調査。産業機器では販売終了品の修正なし事例も EoL資産の棚卸しを「いつか」から期限付き計画へ。除去できない資産はネットワーク分離を唯一の防御策として設計
6 脆弱性対応の官民インフラが再編へ 米政府がAI活用の脆弱性クリアリングハウス構想を始動。国内では重要インフラ向け「パッチ適用の外部サービス化」も発表された 自社の脆弱性運用をこれら外部パイプラインと接続できるよう、資産・SBOM・優先順位付けのデータ整備を先行

推奨アクション（経営判断が必要なもの）

1. 外部公開している基盤製品の「パッチ＋侵害調査」を今週分は特別扱いで完遂させる

文書共有基盤・境界装置・検査アプライアンス・Webサイト構築基盤の今週の欠陥群は、修正公開前から悪用されていたものや、当局の修正期限がすでに到来しているものを含む。適用済みかどうかではなく「侵入されていないか」までの確認を、費用を付けて完了させる（対象と手順は実務者レポート側に整理済み）。

2. 「基幹システム1週間停止」を想定した事業影響の定量化とBCP演習を指示する

国内の同業・近隣業種で物流・生産停止が現実に起きた。停止時損失（1日あたり）・手作業代替の限界・復旧優先順位を数字で持ち、取引先への影響通知の基準と体制まで含めて演習する。上場企業は「恐喝グループが株価を圧力に使う」前提で開示・IR対応も含める。

3. セキュリティ投資の重心をID防御へ再配分する方針を決める

侵入の約8割がID起点というデータが出た。MFAの例外撲滅、ヘルプデスクのコールバック義務化（先週からの継続）、ID脅威検知の導入、そして2027年の旧式MFA廃止期限に向けたパスキー移行計画を、一つのプログラムとして予算化する。

4. AIエージェント製品の利用統制ルールを今期中に確定する

未修正欠陥が公表されている製品の一時停止判断、エージェントへの権限付与の上限、開発者向けAIツールの設定ファイル・認証情報の保護を含む。「便利だから黙認」の状態は、9割の組織がエージェント関連侵害を経験し、63%がポリシー自体を欠くという調査結果と両立しない。

5. 制裁・規制変化への対応を1件ずつ棚卸しする

欧英の対露制裁対象（SNS・メッセンジャー等）のスクリーニング反映、米防衛認証制度の凍結を受けても自主水準を維持する方針の明文化、モバイルOSのAI開放命令（欧州）を踏まえた社用端末統制の見直し、年齢確認法制（世界30超で施行段階）への該当性確認。いずれも実務は軽いが、方針決定は経営マターである。

総評

今週の主戦場は「外部公開された基盤製品」だった。中心は Microsoft SharePoint で、週を通じて 4件が KEV 入り（CVE-2026-32201 / 45659 / 56164 / 58644）し、うち CVE-2026-56164 は Patch Tuesday の悪用確認済みゼロデイ、CVE-2026-58644 は CVSS 9.8 の未認証RCE（CISA 期限 7/19）。さらに JWT 認証バイパス CVE-2026-55040 は連鎖先の RCE 側修正が8月予定のため、パッチを当てても攻撃面が完全には閉じない「未完の連鎖」を翌週に持ち越す。2025年の ToolShell 大量悪用の再来が現実的な懸念になっている。

境界機器では SonicWall SMA1000 のゼロデイ2件（CVE-2026-15409 CVSS 10.0 / CVE-2026-15410）が Inc ランサムウェア関与のアクター（UTA0533）に 6/22 から悪用されていたことが判明。修正公開（7/14）の3週間以上前から侵入されていた計算で、「パッチ適用＝安全」ではなく侵害調査までがワンセットである。FortiSandbox の2件（CVE-2026-39808 / 25089、KEV・連邦期限 7/19）も同型で、7/18 には CVE-2026-39808 が公表から24時間以内に実悪用されたことを Defused Cyber が確認した。

週末の新展開は WordPress コアの事前認証RCEチェーン「wp2shell」（CVE-2026-63030 + CVE-2026-60137）である。修正翌日の 7/17 に PoC 公開、7/18 には watchTowr が実環境での悪用開始の兆候を確認。WordPress.org は強制自動更新を発動し、Cloudflare は全プランに WAF ルールを展開したが、WAF があっても本体パッチは必須。素のインストールでも悪用可能で、推定5億超のサイトが影響範囲に入る。

前週最大の持ち越しだった Progress ShareFile は「パストラバーサルのゼロデイ悪用」と確定し、修正版が出て一応の決着を見た（CVE番号は来週公表見込み）。一方、新たな未解決として、研究者 Nightmare Eclipse（Chaotic Eclipse）が Patch Tuesday 数時間後に Windows の未修正ゼロデイ「LegacyHive」の PoC を公開（CVE未採番・パッチ未提供）。同氏の連続公開は9件目で、過去分（BlueHammer 等）はランサムウェアに悪用された実績があり、短期武器化が警告されている。

量的にも Microsoft の月例修正が過去最多の約620件（2カ月連続記録更新、AI による脆弱性発見が押し上げ）、Mandiant は平均悪用時間がマイナス7日（パッチ公開前に悪用開始）と報告。全量適用は破綻しており、KEV・悪用兆候ベースの優先順位付けが前提になる。

11. 防御アーキテクチャ・セキュリティ製品

※ベンダー発信・自社調査の数値は参考値として扱うこと。

「アイデンティティを制御平面に」が今週の主調。Token Security はローカル AI エージェントの2割超が人間の資格情報を借用して本番データへアクセスし監査ログで区別不能と報告し、人間・AI・マシン ID 統合管理の Oak が6,000万ドルのシードを調達した。7/18 には CrowdStrike が新カテゴリ AIDR（AI Detection and Response） を提唱（ランタイム防御・統合プラットフォーム・実行時制御、エージェント起点検知は人間起点の2.5倍）。防御側の AI 化も加速: Google Cloud の「エージェンティック・ディフェンス」戦略（Wiz 統合）、ソフトバンク×OpenAI の重要インフラ向け「Patching as a Service」、Microsoft の Defender Experts のマルチクラウド拡張。一方で Defender 自体の欠陥（RoguePlanet パッチ疑義）や Bind Link による EDR 回避が出ており、「防御製品を無条件に信頼しない」検証設計が引き続き必要。Cloudflare の WordPress 向け WAF 即日全プラン展開は、CDN/WAF 層の緊急対応が実質的な業界インフラになりつつあることを示した。
